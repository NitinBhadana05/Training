Recovery workflow

Reset instruction lifecycle

01 Request intake

User submits an email address. The system responds safely without revealing account existence.

02 Token issue

A signed reset token is generated with a short expiration window and reset-only token type.

03 Email dispatch

SMTP sends the reset link to the configured recipient mailbox using the app sender profile.

04 Password update

Submitted password must pass policy validation before the account hash is updated in PostgreSQL.
